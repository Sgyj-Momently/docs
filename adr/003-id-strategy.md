# ADR 003 - ID Strategy

## 상태

승인됨

## 날짜

2026-04-20

## 맥락

이 프로젝트는 Spring 오케스트레이터와 여러 에이전트 모듈이 분산 환경으로 확장될 가능성을 전제로 설계하고 있다.
워크플로, 에이전트 작업, 산출물 참조 같은 식별자는 서버 간 중복이 없어야 하며, 향후 높은 트래픽에서도 안정적으로 생성 가능해야 한다.

후보 전략은 아래와 같았다.

- `UUIDv4`
- `ULID`
- `UUIDv7`
- `Snowflake`

## 결정

- 외부 공개 식별자와 워크플로 식별자의 기본 전략은 `UUIDv7`로 한다.
- 내부 DB 전용 숫자 키가 반드시 필요한 경우에만 `Snowflake` 계열 전략을 별도로 검토한다.
- 현재 단계에서는 기본 키 전략을 이원화하지 않고, 공개 식별자 기준을 먼저 `UUIDv7`로 고정한다.

## 이유

- `UUIDv7`은 시간 순 정렬 특성을 가지면서도 표준 기반 식별자라서 장기 유지보수에 유리하다.
- 중앙 조정 서버 없이도 분산 환경에서 생성 가능하다.
- 서버별 worker id 운영, 시계 역행 처리 같은 부담이 `Snowflake`보다 작다.
- 외부 API, HATEOAS 링크, 로그 상관관계 추적, 아티팩트 참조에 폭넓게 사용하기 좋다.
- 현재 프로젝트 단계에서는 운영 복잡도보다 안전한 기본 전략과 표준 친화성이 더 중요하다.

## 결과

- `workflow_id`는 `UUIDv7`을 기준으로 설계한다.
- 추후 DB 성능 검증 결과 숫자형 내부 키가 필요해질 경우, 공개 ID와 내부 PK를 분리하는 방향을 검토한다.
- 스프링 프로젝트 규칙 문서와 오케스트레이터 설계 문서에 이 결정을 반영한다.
