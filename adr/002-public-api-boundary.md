# ADR 002 - Public API Boundary

## 상태

채택

## 배경

그룹화 에이전트 API 요청에 `ollama_base_url` 같은 내부 인프라 설정이 노출되면, 공개 계약과 서버 내부 실행 책임이 섞이게 된다.

## 결정

- 공개 API는 도메인 의도와 데이터만 받는다.
- 내부 모델 서버 주소, timeout 같은 값은 서버 내부 설정으로 관리한다.
- 그룹화 전략은 자유 텍스트가 아니라 enum으로 제한한다.

## 결과

- Spring은 에이전트 API 주소만 알면 된다.
- FastAPI 에이전트가 내부 모델 선택과 주소를 관리한다.
- API 계약이 더 안정적이고 REST 경계에 맞는 구조가 된다.
