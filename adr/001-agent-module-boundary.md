# ADR 001 - Agent Module Boundary

## 상태

채택

## 배경

사진 처리 파이프라인은 여러 단계의 책임으로 나뉜다. 각 단계는 향후 독립 서비스로 분리될 가능성이 있으며, 입력/출력 계약과 API 경계가 중요하다.

## 결정

- 상위 워크스페이스는 하나로 유지한다.
- 각 에이전트는 독립 모듈로 분리한다.
- 각 모듈은 README, 스키마, 테스트, API 명세를 가진다.
- 전체 호출 순서와 상태 관리는 Spring 오케스트레이터가 담당한다.

## 결과

- 모듈 경계가 분명해진다.
- 개별 에이전트를 FastAPI 서비스로 감싸기 쉬워진다.
- 후속 에이전트 추가 시 같은 패턴을 재사용할 수 있다.
